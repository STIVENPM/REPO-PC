¿Cuál es la estructura en HTML?

La estructura de un documento HTML se basa en una jerarquía de etiquetas que organiza el contenido en dos secciones fundamentales dentro de la etiqueta raíz html. La primera es el head, que funciona como el cerebro del documento almacenando metadatos, configuraciones de caracteres, el título de la pestaña y enlaces a recursos externos que no son visibles para el usuario.

La segunda es el body, que constituye el cuerpo visible de la página y donde se declaran todos los elementos estructurales como encabezados, párrafos, imágenes e hipervínculos. Esta arquitectura, encabezada siempre por la declaración !DOCTYPE html, garantiza que el navegador interprete correctamente la estructura del documento.

Tipos de estructuras

Estas etiquetas sirven para organizar la estructura de una página web y se dividen en dos partes:

Estructuras de configuración (El contenedor)

  • html: Es la etiqueta raíz de todo el documento. Representa el inicio y el final de la página web. Todo el contenido del documento debe estar dentro de esta etiqueta.
  • head: Es la sección de configuración del documento. Aquí se guarda información que no es visible directamente para el usuario, como el título de la página, enlaces a archivos CSS o metadatos.

Estructuras de contenido y diseño (Lo visible)

  • body: Es el cuerpo de la página web. Todo lo que pongas dentro de esta etiqueta es lo que el usuario verá en el navegador, como textos, imágenes, botones o enlaces.
  • nav: Se utiliza para definir bloques de navegación. Normalmente contiene menús con enlaces que permiten moverse entre las diferentes páginas del sitio web.
  • section: Define una sección dentro del documento. Se utiliza para agrupar contenido relacionado como secciones de servicios, información o contenido principal.
  • footer: Es el pie de página del sitio web. Generalmente contiene información adicional como derechos de autor, enlaces a redes sociales o información de contacto.
